# Storage Planner

## What this tool does
The Storage Planner estimates how much video storage a camera system may require based on:

- number of cameras
- bitrate per camera
- video retention period
- recording mode
- scene activity
- codec efficiency
- planning overhead reserve

It is designed to provide a more realistic estimate than simple “bitrate × time” storage math by accounting for different recording behaviors.

## When to use it
Use this tool when you need to:

- estimate NVR or server storage capacity
- compare continuous recording to motion or event recording
- plan retention periods
- understand how bitrate and activity affect storage use
- prepare a rough system design before selecting hardware

## Inputs

### Number of Cameras
The total number of cameras that will be recording to the storage system.

### Estimated Bitrate per Camera (Mbps)
The average data rate generated by one camera.

Higher bitrate usually means:

- better image quality
- larger storage requirements
- higher network load

If the exact bitrate is unknown, use the Bitrate Estimator first.

### Video Retention (Days)
How long recorded video should remain available before it is overwritten.

Example:
- `30` means video is kept for 30 days

### Video Codec
The compression method used by the camera.

Common options:
- **H.264** — older, less efficient
- **H.265** — more efficient than H.264
- **H.265+ / Smart Codec** — often more efficient in lower-motion scenes

Codec choice affects how much storage is required for the same scene.

### Recording Mode
Defines when the system records video.

- **Continuous Recording** — records all day, every day
- **Motion Recording** — records only when motion is detected
- **Event / AI Recording** — records only when an event or analytic trigger occurs

### Scene Activity Preset
Provides common starting values for how active a scene is.

Examples:
- Quiet Perimeter
- Office Hallway
- Warehouse
- Parking Lot
- Retail Sales Floor
- Busy Entrance

These presets auto-fill an activity percentage.

### Activity Percentage
The estimated percentage of the day that the camera is actively recording meaningful motion or events.

Examples:
- quiet scene = lower percentage
- busy public area = higher percentage

This value is especially important for motion and event-based recording.

### Overhead Reserve
A conservative planning buffer that accounts for:

- bitrate spikes
- metadata
- filesystem overhead
- real-world variance

Typical reserve:
- **10% to 25%**

## Results

### Estimated Storage Required
The total estimated storage required for the selected system over the retention period.

### Per-Camera Storage per Day
The estimated storage consumed by one camera in one day.

### Total Storage per Day
The estimated storage consumed by all cameras in one day.

### Recording Mode Comparison
Shows how total storage changes if the same system uses:

- continuous recording
- motion recording
- event / AI recording

This helps compare design choices more realistically.

### Data Summary
Shows:
- per camera / hour
- per camera / day
- total daily storage

## Notes
This planner uses GiB/TiB-style math based on 1024.

Real-world results vary depending on:

- codec tuning
- scene complexity
- lighting conditions
- frame rate
- resolution
- keyframe interval
- VBR behavior
- analytics settings

Use this as a planning tool, not a manufacturer-certified guarantee.
