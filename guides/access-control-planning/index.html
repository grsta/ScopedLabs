Use the tools as a linear planning workflow.

Access control planning usually starts with how each opening should behave, then moves into reader choice, lock power, controller capacity, and access level structure. ScopedLabs keeps that process practical by turning each step into a focused calculator or planning aid.

These tools are planning aids only. They help organize assumptions and expose design pressure early, but they do not replace code review, AHJ direction, manufacturer documentation, or final engineering review.

The Access Control flow

The public Access Control design flow is a straight step-by-step lane:

ACCESS CONTROL FLOW
Fail-Safe / Fail-Secure → Reader Type → Lock Power → Panel Capacity → Access Levels

1. Decide fail-state behavior first

Start with whether the opening should unlock or stay locked during power loss. This decision affects lock type, fire-life-safety coordination, battery planning, and how the opening behaves during an outage.

Open Fail-Safe / Fail-Secure

2. Select the reader approach

Reader choices are shaped by security level, environment, throughput, credential preference, and the controller interface. A reader that works on paper can still create usability or compatibility issues later.

Open Reader Type Selector

3. Check lock power pressure

Lock power planning should account for simultaneous unlock events, supply capacity, voltage behavior, and practical headroom. This is where undersized power supplies and unstable circuits often show up.

Open Lock Power Budget

4. Size panel and expansion capacity

Panel capacity is more than counting doors. Spare inputs, outputs, expansion modules, future growth, and controller limits all influence whether the system has enough room to grow cleanly.

Open Panel Capacity

5. Keep access levels manageable

Access level structure can become hard to administer if roles, schedules, exceptions, and door groups grow without a plan. The goal is not just security; it is a structure people can maintain accurately.

Open Access Level Sizing

6. Use supporting calculators when needed

Door count, credential format, anti-passback, elevator readers, and cable length checks can support the main design when those constraints matter to the project. Use them to validate assumptions before the final plan.

What to document before calling the design “done”

Door behavior: Which openings are fail-safe, fail-secure, or require special coordination.
Reader assumptions: Reader type, interface, credential preference, environment, and expected traffic.
Power assumptions: Lock current, simultaneous unlock count, supply headroom, and backup expectations.
Panel capacity: Controlled openings, spare capacity, expansion needs, and growth margin.
Access structure: Roles, schedules, access groups, exceptions, and administrative complexity.
