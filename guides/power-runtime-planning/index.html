Planning workflow

What power and runtime planning means

Power planning is not just picking a UPS size from a nameplate. A practical estimate starts with the connected load, converts that load into compatible electrical units, adds realistic growth margin, then checks whether the backup system can support the required runtime without operating too close to its limits.

ScopedLabs treats power and runtime planning as a sequence: estimate the load, normalize the units, account for growth, review runtime, and document the assumptions. That makes the result easier to explain, compare, and revisit later.

Why watts, VA, amps, and power factor all matter

Power equipment is often described with different units. A device might list watts, a UPS might be rated in VA, and a branch circuit might be limited by amps. Power factor connects real power and apparent power, so ignoring it can make an estimate look cleaner than the actual electrical load.

Use the VA / Watts / Amps Calculator when you need to normalize load information before moving into runtime, battery, or branch planning. If inverter losses are part of the design, the Inverter Efficiency Calculator helps separate usable output from conversion overhead.

A practical planning sequence

1. Estimate the connected load

List the equipment that must stay online, estimate its operating draw, and convert the load into consistent units. This becomes the baseline for the rest of the power plan.

Load baseline
2. Add growth and operating margin

Plan for near-term additions, replacement hardware, and reasonable spare capacity instead of sizing only for today. Load Growth Planner is the right next step when you need to explain future headroom.

Headroom
3. Estimate UPS runtime

Runtime depends on load, battery capacity, efficiency, and how aggressively the system is being used. UPS Runtime Calculator should be treated as a planning estimate, not a manufacturer runtime guarantee.

Runtime
4. Review battery sizing

Battery sizing should consider target runtime, usable capacity, efficiency, aging, and whether the system has enough reserve for real-world conditions. Use the Battery Sizing Calculator to check battery assumptions.

Battery reserve
5. Check branch and loading assumptions

A runtime estimate is only useful if the connected electrical path can support the load. Branch Planning Calculator helps keep branch planning separate from battery/runtime assumptions when that calculator is available.

Electrical path

Common planning mistakes

Using nameplate load as the final answer

Nameplate values can be useful as a boundary, but planning should distinguish estimated operating load from worst-case label values.

Forgetting future load growth

A UPS that barely fits today may become a problem after one equipment refresh, added switch, larger NVR, or extra compute node.

Treating runtime as a fixed number

Runtime changes as the load changes. A small load increase can have an outsized effect when the battery system is already near its useful limit.

Mixing units without checking power factor

Watts, VA, and amps are related, but they are not interchangeable without voltage and power factor assumptions.
